# Governance Checklist

## Purpose

Use this checklist to confirm that multi-cloud governance controls are in place before major platform changes are approved.

## Checklist

### Policy

- [ ] Policy owner identified
- [ ] Policy version current
- [ ] Exception process defined

### Landing Zone

- [ ] Baseline approved
- [ ] Identity configured
- [ ] Logging enabled
- [ ] Network segmentation applied

### Review

- [ ] Architecture review completed
- [ ] Approval recorded
- [ ] Conditions tracked

### Compliance

- [ ] Control mapping complete
- [ ] Evidence location known
- [ ] Audit gaps documented

### Drift

- [ ] Drift baseline defined
- [ ] Detection method active
- [ ] Remediation owner assigned
